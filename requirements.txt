# Audited 2026-07-04 (see CLAUDE.md's dependency guardrail): CP-SAT
# (ortools.sat.python.cp_model) is a fully local native solver — no network
# calls. The only network-capable code in the package is the opt-in
# ortools.math_opt.python.ipc.remote_http_solve helper, which POSTs to
# https://optimization.googleapis.com and requires an explicit API key to
# invoke. It is unrelated to CP-SAT and this codebase must never import or
# call it.
ortools>=9.0

# Audited 2026-07-06: Streamlit defaults to phoning home anonymous usage
# stats (browser.gatherUsageStats=True by default, POSTs are gated by that
# exact config flag in streamlit/runtime/metrics_util.py) and, on first run
# with no local credentials file, interactively prompts for an email address
# that — only if submitted — gets POSTed for a mailing list. Both are
# disabled in .streamlit/config.toml (gatherUsageStats=false, headless=true,
# which together fully skip both code paths per
# streamlit/runtime/credentials.py's check_credentials()). Do not remove
# that config file.
streamlit>=1.59

# Audited 2026-07-13: the `ollama` PyPI package (checked v0.6.2) is a thin
# httpx wrapper — every request goes to the single `host` passed to
# Client()/AsyncClient(), defaulting to http://127.0.0.1:11434
# (ollama/_client.py:_parse_host). No other network code in the package.
#
# The Ollama *server* itself (separate Homebrew install, not this package)
# ships a cloud feature (remote inference + web search against ollama.com,
# env vars OLLAMA_REMOTES/OLLAMA_NO_CLOUD) as of v0.31.1 — this is an
# outbound path that violates CLAUDE.md's no-cloud-inference constraint
# unless explicitly disabled. The server MUST be started with
# OLLAMA_NO_CLOUD=1 (llm/client.py checks this at import time and refuses
# to run otherwise) and models pulled must be plain local tags (e.g.
# "llama3.1:8b"), never a "-cloud" suffixed tag, which routes to
# ollama.com regardless of OLLAMA_NO_CLOUD.
ollama>=0.6.0
